OctoAcme Project Management Docs
Overview
Welcome to OctoAcme's project management documentation hub. This folder contains our core project management processes, templates, and guidance for running projects from initiation through retrospectives. Whether you're a new team member, a project manager, product lead, or developer, these docs will help you understand how OctoAcme approaches project delivery.

What is OctoAcme?
OctoAcme is a customer-first, iteratively-driven project management framework built on principles of clear ownership, data-informed decisions, and psychological safety. Our approach emphasizes delivering small, testable increments with clear success metrics and continuous improvement.

How to Use These Docs
Each document in this folder covers a specific phase of the project lifecycle:

Start here: Read the Project Management Overview for a high-level introduction to roles, lifecycle, and key artifacts.
Find your phase: Use the Docs Index below to navigate to the specific process you need.
Reference templates: Each doc includes checklists, templates, and examples you can adapt for your project.
Ask questions: If something is unclear, open an issue in the repository with the "documentation" label.
OctoAcme Project Management Lifecycle
Our project management process is built on 7 core phases:

Phase	Purpose	Key Deliverable
Initiation	Validate business need, align stakeholders, define success metrics	Project One-pager
Planning	Break work into shippable increments, identify risks and dependencies	Prioritized Backlog & Release Plan
Execution & Tracking	Build, test, review, and iterate with daily team rhythm and quality gates	Working Software & Progress Metrics
Risk & Communication	Identify and mitigate risks, keep stakeholders informed	Risk Register & Status Updates
Release & Deployment	Standardize how we move features to production safely	Release Notes & Deployment Verification
Retrospective & Improvement	Capture learnings and convert them into actionable improvements	Action Items & Process Updates
Roles & Personas	Define key roles and responsibilities across delivery teams	Role Definitions & Communication Patterns
Core Principles
Customer-first: Prioritize customer value and usability in every decision
Iterative delivery: Deliver small, testable increments with regular feedback loops
Clear ownership: Every project has named PM and Product Lead; every task has clear ownership
Data-informed decisions: Measure impact and iterate based on evidence
Psychological safety: Encourage feedback, learning, and blameless retrospectives
Key Roles
Project Manager (PM): Coordinates delivery, manages schedules, risks, and communications
Product Manager (PdM): Defines outcomes, prioritizes backlog, measures success
Developers: Design, build, test, and deliver features
QA/Testing: Validate quality and acceptance criteria
Stakeholders: Provide inputs, approvals, and strategic direction
See Roles and Personas for detailed definitions.

Quick Start Checklist
Just starting a new project? Use this checklist:

 Read the Project Management Overview
 Follow Project Initiation to create a One-pager and align stakeholders
 Move to Project Planning once initiation is approved
 Use Execution and Tracking for your daily workflow
 Reference Risks and Communication for ongoing risk management
 Plan your release using Release and Deployment
 Schedule a retrospective and follow Retrospective and Continuous Improvement
Communication Cadence
Daily: Team standups (15 min) — progress, blockers, dependencies
Weekly: PM + PdM sync, delivery team check-in, risk review
Bi-weekly/Monthly: Stakeholder updates and demos
As needed: Escalations, incident responses, decision gates
Frequently Asked Questions
Q: Which document should I read first?
A: Start with Project Management Overview for a high-level view, then navigate to the phase-specific doc for your current project stage.

Q: Can I adapt these processes for my team?
A: Yes! These docs provide a foundation. Work with your team and stakeholders to adapt checklists, meeting cadences, and templates to your context.

Q: How do I propose updates or improvements to these docs?
A: Open an issue using the "Add Content to Project Management Process Docs" template (.github/ISSUE_TEMPLATE/add-update-content-to-process-docs.yml). We encourage continuous improvement!

Q: Where are templates and examples?
A: Each phase document includes templates (One-pager, Risk Register, Status Update, etc.) and checklists you can copy and customize.
